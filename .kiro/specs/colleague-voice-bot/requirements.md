# Requirements Document

## Introduction

The Colleague Voice Bot is a fun, interactive AWS-hosted application that clones the voices and accents of 7 colleagues from short voice recordings. Users can type or speak text and hear it played back in any colleague's voice. The system is designed to be entertaining — think of it as a party trick: generate fake "quotes" from colleagues, run a quiz to guess who said what, or let colleagues roast each other in their own voices. Everything runs on AWS in us-east-1.

## Glossary

- **Voice_Bot**: The overall application system described in this document.
- **Voice_Profile**: A trained voice model derived from one colleague's recordings, used to synthesize speech in that colleague's voice.
- **Voice_Sample**: An audio recording of a colleague used as training input for a Voice_Profile.
- **Synthesized_Speech**: Audio output generated by the Voice_Bot in a specific colleague's voice.
- **Voice_Quiz**: A game mode where the user listens to Synthesized_Speech and guesses which colleague's voice was used.
- **Quote_Generator**: A feature that produces humorous or random text attributed to a colleague and speaks it in their voice.
- **Admin**: The person responsible for uploading Voice_Samples and managing Voice_Profiles.
- **User**: Any person interacting with the Voice_Bot's playback and game features.
- **S3**: Amazon Simple Storage Service, used for storing Voice_Samples and Synthesized_Speech audio files.
- **Voice_Cloning_Service**: A deep learning voice cloning model (e.g. Coqui XTTS v2 or equivalent open-source model) hosted on an AWS SageMaker endpoint, responsible for generating Synthesized_Speech from a Voice_Profile while preserving accent and vocal characteristics.
- **Singing_Mode**: A feature that synthesizes a short phrase or lyric as sung audio in a colleague's voice rather than spoken speech.
- **Language_Mode**: A feature that synthesizes text in a target language while preserving the colleague's voice and accent characteristics.
- **API_Gateway**: AWS API Gateway, the HTTP entry point for all Voice_Bot interactions.
- **Lambda**: AWS Lambda functions that implement Voice_Bot business logic.
- **DynamoDB**: AWS DynamoDB, used to store Voice_Profile metadata and quiz scores.
- **CloudFront**: AWS CloudFront distribution serving the Voice_Bot web UI and audio files.

## Requirements

### Requirement 1: Voice Sample Ingestion

**User Story:** As an Admin, I want to upload voice recordings for each of my 7 colleagues, so that the system has enough material to build accurate Voice_Profiles.

#### Acceptance Criteria

1. THE Voice_Bot SHALL accept audio file uploads in MP3, WAV, and M4A formats for each colleague.
2. WHEN an Admin uploads a Voice_Sample, THE Voice_Bot SHALL store the file in S3 under a key namespaced by colleague identifier.
3. WHEN an uploaded file is smaller than 10 seconds of audio, THE Voice_Bot SHALL reject the upload and return a descriptive error message indicating the minimum duration requirement.
4. WHEN an uploaded file exceeds 300 seconds of audio, THE Voice_Bot SHALL reject the upload and return a descriptive error message indicating the maximum duration limit.
5. THE Voice_Bot SHALL support uploading between 1 and 10 Voice_Samples per colleague.
6. WHEN a Voice_Sample upload succeeds, THE Voice_Bot SHALL return a confirmation response containing the assigned sample identifier and the colleague identifier.

---

### Requirement 2: Voice Profile Creation

**User Story:** As an Admin, I want the system to build a Voice_Profile for each colleague from their uploaded recordings, so that the Voice_Bot can synthesize speech in that colleague's voice.

#### Acceptance Criteria

1. WHEN at least one Voice_Sample exists for a colleague, THE Voice_Bot SHALL allow the Admin to trigger Voice_Profile creation for that colleague.
2. WHEN Voice_Profile creation is triggered, THE Voice_Bot SHALL submit the associated Voice_Samples to the Voice_Cloning_Service SageMaker endpoint and store the resulting Voice_Profile metadata in DynamoDB.
3. WHEN Voice_Profile creation completes successfully, THE Voice_Bot SHALL update the colleague's status in DynamoDB to "ready".
4. IF the Voice_Cloning_Service returns an error during Voice_Profile creation, THEN THE Voice_Bot SHALL update the colleague's status in DynamoDB to "failed" and store the error details.
5. WHILE Voice_Profile creation is in progress, THE Voice_Bot SHALL report the colleague's status as "processing" in DynamoDB.
6. THE Voice_Bot SHALL support Voice_Profiles for exactly 7 colleagues simultaneously.

---

### Requirement 3: Speech Synthesis

**User Story:** As a User, I want to type any text and hear it spoken back in a chosen colleague's voice, so that I can generate funny or creative audio clips.

#### Acceptance Criteria

1. WHEN a User submits text and a colleague identifier, THE Voice_Bot SHALL synthesize the text using the corresponding Voice_Profile and return an audio URL.
2. WHEN synthesis is requested for a colleague whose Voice_Profile status is not "ready", THE Voice_Bot SHALL return an error response indicating the profile is unavailable.
3. THE Voice_Bot SHALL accept input text between 1 and 500 characters per synthesis request.
4. WHEN input text exceeds 500 characters, THE Voice_Bot SHALL return a descriptive error message indicating the character limit.
5. WHEN synthesis completes, THE Voice_Bot SHALL store the Synthesized_Speech audio file in S3 and return a pre-signed URL valid for 24 hours.
6. WHEN the same text and colleague identifier are submitted within a 1-hour window, THE Voice_Bot SHALL return the cached audio URL from S3 rather than re-synthesizing.

---

### Requirement 4: Quote Generator

**User Story:** As a User, I want the system to generate and speak a random humorous quote in a colleague's voice, so that I can enjoy a fun, shareable audio clip without typing anything.

#### Acceptance Criteria

1. WHEN a User requests a random quote for a colleague, THE Voice_Bot SHALL select a quote from a pre-loaded quote library and synthesize it using that colleague's Voice_Profile.
2. THE Voice_Bot SHALL maintain a quote library of at least 50 quotes covering topics such as office life, meetings, and technology.
3. WHEN a User requests a random quote, THE Voice_Bot SHALL select a quote that was not used in the previous 5 requests for the same colleague, to avoid immediate repetition.
4. WHEN a User requests a random quote for a colleague whose Voice_Profile status is not "ready", THE Voice_Bot SHALL return an error response indicating the profile is unavailable.
5. WHEN a quote is synthesized, THE Voice_Bot SHALL return both the quote text and the audio URL so the User can read along.

---

### Requirement 5: Voice Quiz Game

**User Story:** As a User, I want to play a guessing game where I hear a synthesized clip and try to identify which colleague's voice was used, so that I have a fun, interactive experience.

#### Acceptance Criteria

1. WHEN a User starts a Voice_Quiz, THE Voice_Bot SHALL select a random colleague with a "ready" Voice_Profile and synthesize a random quote in that colleague's voice as either a spoken or sung clip.
2. WHEN a Voice_Quiz round is presented, THE Voice_Bot SHALL provide the User with the names of all 7 colleagues as answer options.
3. WHEN a Voice_Quiz round is presented, THE Voice_Bot SHALL indicate whether the clip is a spoken round or a Singing_Mode round so the User knows the format.
4. WHEN a User submits a guess, THE Voice_Bot SHALL evaluate the guess against the correct colleague identifier and return a result indicating correct or incorrect.
5. WHEN a User submits a correct guess, THE Voice_Bot SHALL increment that User's score in DynamoDB by 1 point.
6. WHEN a User submits an incorrect guess, THE Voice_Bot SHALL reveal the correct colleague's name in the response.
7. THE Voice_Bot SHALL track a leaderboard of the top 10 User scores in DynamoDB, ordered by total correct guesses descending.
8. WHEN a User submits a score to the leaderboard, THE Voice_Bot SHALL record the score under a nickname provided by the User, without requiring authentication.
9. WHEN a User requests the leaderboard, THE Voice_Bot SHALL return the top 10 entries including nickname and score.

---

### Requirement 6: Language Mode

**User Story:** As a User, I want to synthesize text in a language other than English while still using a colleague's voice, so that I can generate funny multilingual clips that preserve the colleague's accent and vocal characteristics.

#### Acceptance Criteria

1. WHEN a User submits a synthesis request with a target language code (e.g. "en", "fr", "hi"), THE Voice_Bot SHALL synthesize the text in that language using the colleague's Voice_Profile.
2. THE Voice_Bot SHALL support exactly the following target languages: English (en), French (fr), and Hindi (hi).
3. WHEN a target language is specified, THE Voice_Bot SHALL pass the language parameter to the Voice_Cloning_Service SageMaker endpoint alongside the Voice_Profile and input text.
4. WHEN a target language code is submitted that is not in the supported language list, THE Voice_Bot SHALL return a descriptive error message listing the supported language codes.
5. WHEN Language_Mode synthesis completes, THE Voice_Bot SHALL store the Synthesized_Speech audio file in S3 and return a pre-signed URL valid for 24 hours, following the same caching rules as standard synthesis.

---

### Requirement 7: Singing Mode

**User Story:** As a User, I want to make a colleague "sing" a short phrase or lyric in their own voice, so that I can generate entertaining sung audio clips and use them in a quiz variant.

> **Note on Voice_Sample source:** Voice_Profiles are built exclusively from spoken meeting recordings. No dedicated singing recordings are required or expected. Singing_Mode synthesis is performed by the Voice_Cloning_Service using the same spoken-audio Voice_Profile; sung output quality is best-effort and may vary.

#### Acceptance Criteria

1. WHEN a User submits a synthesis request with the singing mode flag set, THE Voice_Bot SHALL synthesize the input text as sung audio using the specified colleague's Voice_Profile, which is derived from spoken meeting recordings.
2. THE Voice_Bot SHALL accept input text for Singing_Mode between 1 and 200 characters per request.
3. WHEN input text for Singing_Mode exceeds 200 characters, THE Voice_Bot SHALL return a descriptive error message indicating the character limit for sung phrases.
4. WHEN Singing_Mode synthesis completes, THE Voice_Bot SHALL store the Synthesized_Speech audio file in S3 and return a pre-signed URL valid for 24 hours.
5. WHEN a User requests a Voice_Quiz round in singing variant mode, THE Voice_Bot SHALL synthesize the quiz clip using Singing_Mode and present the same colleague-guessing challenge as a standard quiz round.
6. WHEN a Voice_Quiz singing round is presented, THE Voice_Bot SHALL label the clip as a singing round so the User is aware of the increased difficulty.
7. WHEN the web UI presents a Singing_Mode synthesis result generated from a spoken-only Voice_Profile, THE Voice_Bot SHALL display a disclaimer to the User stating that singing output is experimental and quality may vary when using spoken-only profiles.

---

### Requirement 8: Web User Interface

**User Story:** As a User, I want a simple, fun web interface to interact with all Voice_Bot features, so that I can use the application without any technical knowledge.

#### Acceptance Criteria

1. THE Voice_Bot SHALL serve a web UI via CloudFront that provides access to speech synthesis, Language_Mode, Singing_Mode, the Quote_Generator, and the Voice_Quiz.
2. WHEN a User selects a colleague in the web UI, THE Voice_Bot SHALL display the colleague's name and availability status.
3. WHEN Synthesized_Speech audio is ready, THE Voice_Bot SHALL enable an in-browser audio player so the User can listen without downloading the file.
4. THE Voice_Bot SHALL display a visual indicator while synthesis or Voice_Profile creation is in progress.
5. WHEN a Voice_Quiz answer is submitted, THE Voice_Bot SHALL animate the result (correct or incorrect) to make the experience engaging.
6. THE Voice_Bot SHALL be accessible from modern desktop and mobile browsers without requiring any software installation.
7. WHEN a User submits a score to the leaderboard via the web UI, THE Voice_Bot SHALL prompt the User to enter a nickname before recording the score.

---

### Requirement 9: Security and Access Control

**User Story:** As an Admin, I want upload and profile management actions to be protected, so that only authorized users can modify Voice_Profiles or upload Voice_Samples.

#### Acceptance Criteria

1. THE Voice_Bot SHALL require Admin authentication via AWS Cognito for all Voice_Sample upload, Voice_Profile management, and leaderboard administration endpoints.
2. WHEN an unauthenticated request is made to an Admin endpoint, THE API_Gateway SHALL return an HTTP 401 response.
3. WHEN an authenticated non-Admin user attempts to access an Admin endpoint, THE API_Gateway SHALL return an HTTP 403 response.
4. THE Voice_Bot SHALL allow regular Users to access synthesis, quiz, and leaderboard read/write endpoints without requiring authentication.
5. THE Voice_Bot SHALL store all Voice_Samples and Synthesized_Speech in S3 buckets with public access blocked at the bucket level.
6. WHEN a User requests a Synthesized_Speech audio file, THE Voice_Bot SHALL serve the file via a pre-signed S3 URL rather than a public URL.

---

### Requirement 10: Audio Round-Trip Integrity

**User Story:** As an Admin, I want to verify that uploaded Voice_Samples are stored and retrievable without corruption, so that Voice_Profile quality is not degraded by storage issues.

#### Acceptance Criteria

1. WHEN a Voice_Sample is uploaded and stored in S3, THE Voice_Bot SHALL make the file retrievable via its assigned S3 key.
2. FOR ALL Voice_Samples uploaded to S3, THE Voice_Bot SHALL store a checksum alongside the file metadata in DynamoDB.
3. WHEN a Voice_Sample is retrieved for Voice_Profile creation, THE Voice_Bot SHALL verify the retrieved file's checksum matches the stored checksum before submitting it to the Voice_Cloning_Service.
4. IF a checksum mismatch is detected, THEN THE Voice_Bot SHALL reject the Voice_Profile creation request and return an error indicating data integrity failure.
